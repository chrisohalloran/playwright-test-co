ptc — Playwright Test Co

Run only the Playwright tests affected by your PR changes. Full confidence, fraction of the time.

$ ptc -p ./my-app -f src/components/login-form.js

Changed files detected:

Matched tests:
  • tests/login.spec.ts  [import] ← src/components/login-form.js

Running 1 targeted test file(s)... (skipping ~4 unaffected)

  ✓ shows login form with email and password fields (240ms)
  ✓ requires email field to be filled (129ms)

  ✓ navigates to dashboard on successful login (588ms)

  4 passed (1.7s)

✓ All targeted tests passed (2.5s, 1/5 test files)

Why targeted testing?

You fix a component in your PR. Your full E2E suite takes 10+ minutes. You only changed one file. Why run everything?

ptc analyzes which files changed, maps them to the Playwright tests that exercise those areas, and runs only those tests.

Import tracing

Parses test files for imports that reference your changed source files.

Name matching

Fuzzy matches file stems: login-form.js maps to login.spec.ts.

Content reference

Searches test content for mentions of changed modules.

CI-native

Pipe git diff directly. Works with any CI. GitHub Action included.

Install

npm (global) npm install -g playwright-test-co
npx (no install) npx playwright-test-co -p ./my-app --git-base main
GitHub Action - uses: chrisohalloran/playwright-test-co@v1

GitHub Action — one line

Add targeted Playwright testing to any PR workflow:

workflow name: Targeted E2E Tests
on: pull_request
jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
      - run: npm ci
      - uses: chrisohalloran/playwright-test-co@v1
